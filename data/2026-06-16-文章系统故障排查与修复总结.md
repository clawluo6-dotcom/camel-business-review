# 文章系统故障排查与修复总结

> 日期：2026-06-16
> 涉及项目：骆驼商业本质静态网站

## 概述

今日连续排查并修复了文章系统的 4 类问题，涉及 URL 编码、JS 数据文件损坏、文章内容重复、以及数据 ID 引号不匹配。以下逐一记录问题根因与修复方案。

---

## 问题一：含 `+` 号的文章 ID 点击时变为空格

**现象**：从分类页面点击标题含 `+` 的文章（如《武则天登顶核心博弈逻辑：借矛盾+绑利益》），跳转后提示"文章未找到"。

**根因**：`category.html` 及 11 个其他页面在生成文章链接时，使用模板字面量 `${a.id}` 直接拼接，未对 `a.id` 进行 `encodeURIComponent()` 编码。而 `+` 在 URL 查询参数中会被浏览器解析为空格，导致 `URLSearchParams.get('id')` 拿到的 ID 中的 `+` 变成了空格，与 `site-data.js` 中存储的 ID 不匹配。

**已修复文件（12个）**：
- `category.html`
- `social.html`、`worldview.html`、`practice.html`
- `confucianism-research.html`、`buddhism-research.html`、`traditional-philosophy.html`
- `science-philosophy.html`、`international-finance.html`、`macroeconomic-research.html`
- `china-economy.html`、`business-model.html`、`economic-history.html`

**教训**：凡是把用户数据（文章 ID）嵌入 URL 时，必须使用 `encodeURIComponent()` 进行编码，否则特殊字符（`+`、`&`、`#`、`%`、中文等）会破坏 URL 解析。

---

## 问题二：文章 "03-世界的底层逻辑" 排版溢出

**现象**：文章正文末尾出现一大段无格式的纯文本，没有段落换行，溢出底色边框。

**根因**：`article-content.js` 中该文章的 JS 字符串值末尾被重复拼接了一段纯文本版本的副本（无 Markdown 标记），与前面的 Markdown 正文重复。该副本无 `#` 标题、无 `-` 列表、无 `>` 引用等格式符号，故渲染后所有文字连成一片。

**修复**：从 `data/articles/` 目录下的 JSON 源文件中读取正确的 1588 字内容，替换 `article-content.js` 中被损坏的 3071 字条目。

---

## 问题三：文章批量丢失（修复脚本意外破坏数据文件）

**现象**：`article-content.js` 被错误修复后，仅剩 1 篇文章，其余 156 篇全部丢失，所有文章页面显示"加载中..."。

**根因**：自动修复脚本在替换条目时，未正确处理 JavaScript 字符串中的换行转义，导致写入的文件只包含了被替换的 1 篇文章。

**修复过程**：
1. 从 git 父版本恢复完整文件（约 1.9MB）
2. 发现该版本本身存在 JS 语法错误（文章 ID 含未转义的 Unicode 花引号）
3. 最终方案：直接从 `data/articles/*.json`（167 个 JSON 源文件）重新生成 `article-content.js`，确保所有特殊字符正确转义

**教训**：直接修改大型 JS 数据文件极不安全。正确的做法是从源数据（JSON 文件）重新生成，而非在原文件上 patch。

---

## 问题四：世界观文章 05/07 内容为空（引号字符不匹配）

**现象**：「世界观」→「（四）、《世界的真相》感悟」分类下，文章 05 和 07 点开后内容为空。

**根因**：同一个文章 ID 中的引号字符不一致：

| 数据来源 | 使用字符 | Unicode | 示例 |
|---------|---------|:-------:|:----:|
| `site-data.js` | **花引号** `""` | U+201C / U+201D | `理解"世界是假的"` |
| `data/*.json` | **直引号** `""` | U+0022 | `理解"世界是假的"` |

两种引号视觉上几乎相同，但计算机视为不同字符。`article.html` 使用 `site-data.js` 中的 ID 去 `__ARTICLE_CONTENT__` 中查找时，因为字符不匹配，返回 `undefined`。

**修复**：将 05 和 07 数据文件的 ID 中的直引号替换为花引号，与 `site-data.js` 保持一致，然后重新生成 `article-content.js`。

**教训**：在文本处理中，中文引号（花引号 `""`）和英文引号（直引号 `""`）是不同字符，混用会导致字符串匹配失败。生成数据时应统一使用一种风格。

## 预防措施

1. 所有文章内容以 `data/articles/*.json` 为唯一数据源，`site-data.js` 和 `article-content.js` 由脚本自动生成
2. 生成脚本需正确处理特殊字符编码（花引号、直引号、`+` 号等）
3. HTML 模板中的文章链接统一使用 `encodeURIComponent()` 编码
4. 每次生成后运行 JS 语法检查和内容完整性验证
